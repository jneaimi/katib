Before you start

  • Have access to the system or tool being documented
  • Know your way around basic navigation
  • About 5 minutes of uninterrupted time

Steps

  1. Open the target screen

    Navigate to the page where you'll perform the action. If you're using Soul Hub, this is usually the Vault tab.

  2. Locate the entry point

    Look for the primary action button — typically top-right, labeled with the verb you want to perform.

  3. Fill in the details

    Provide the required fields. Optional fields are marked with a neutral label; required ones have a subtle accent color.

  4. Confirm and verify

    After submitting, you should see the new item appear in the list within a second. If it doesn't, refresh and check for error states.

Tip

Keyboard shortcut ⌘K opens a quick-create dialog on every page — often faster than clicking through the UI.

Watch out

If you see a red banner, the action wasn't saved. Re-read the error message — it usually points to a specific field.

Do & Don't

Do Use descriptive names. Search becomes powerful when names carry meaning — "client-onboarding-v2" beats "doc-final-final".
Don't Reuse names across categories. Two items called "notes" in different folders will collide in search results.

If it doesn't work

  • Page doesn't load: check that the service is running at the expected address.
  • Button greyed out: usually means a prerequisite isn't met — hover for tooltip.
  • Changes don't persist: verify you have write permission to the target folder.
