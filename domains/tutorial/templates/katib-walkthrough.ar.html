كاتب هو أداة إنتاج مستندات بجودة طباعية، ثنائية اللغة (عربي + إنجليزي)، تعمل داخل Claude Code. يأخذك هذا الدليل من التثبيت الأول وحتى حلقة التحسّن الذاتي، في ستّ وحدات مرتّبة.

ما ستتعلّمه
  • تثبيت كاتب وإنتاج أوّل ملف PDF في أقل من خمس دقائق.
  • اختيار المجال المناسب ونوع المستند والغلاف والتنسيق لأيّ تسليم.
  • تطبيق هوية علامتك الشخصية لتبدو كل المستندات كأنها لك.
  • دمج مخطّطات ولقطات شاشة ورسوم بيانية مدمجة داخل الصفحة، بجودة طباعية.
  • تمكين كاتب من التعلّم من تصويباتك عبر أداة reflect.py.

قبل أن تبدأ

  • نظام macOS أو Linux مع Python 3.10 أو أحدث، وأداة uv مثبّتة.
  • Claude Code مُثبّت، مع تسجيل هذه المهارة في المسار ~/.claude/skills/katib/.
  • مجلد Obsidian (أو أي مجلد تختاره) كجذر للمخرجات.
  • اختياري: GEMINI_API_KEY لإنتاج الأغلفة التصويرية (~0.12 دولار للغلاف الواحد).
الوحدة 1 · الأساسيات

التثبيت والإنتاج الأوّل

خمس دقائق من الصفر حتى ملف PDF يحمل هويتك. ستثبّت المهارة، تضبط مسارات الإخراج، وتنتج مستنداً من صفحة واحدة — لتكون لديك إشارة قاطعة بأن خط الإنتاج يعمل كاملاً.

أهداف الوحدة
  • تثبيت كاتب عبر المثبّت المرفق أو عبر npm.
  • إنشاء إعدادات المستخدم في ~/.config/katib/config.yaml.
  • إنتاج عيّنة المستند المكوّن من صفحة واحدة ثم فحص مجلد المخرجات.

الخطوات

  1. ثبّت المهارة

    يأتي كاتب بطريقتين للتثبيت: يمكنك استنساخ المستودع وتشغيل المثبّت المرفق، أو التثبيت مباشرة عبر npm إن كنت تفضّل أمراً واحداً.

    # الخيار أ — المثبّت المرفق
git clone https://github.com/jneaimi/katib.git
cd katib && bash install.sh

# الخيار ب — عبر npm
npm i -g @jasemal/katib && katib install

  2. اضبط المخرجات والهوية

    انسخ ملف الإعدادات النموذجي، وأخبر كاتب أين يضع المستندات المولّدة. المفتاح memory.location هو المكان الذي تقرأ منه أداة reflect.py سجلّاتها لاحقاً.

    # ~/.config/katib/config.yaml
output:
  destination: vault
  vault_path: ~/vault/content/katib

memory:
  location: ~/.local/share/katib/memory

identity:
  author_name: "Jasem Al Neaimi"
  company: "TridenITS"

  3. أنتج أوّل مستند

    أنتج مستنداً من نوع عرض تجاري على صفحة واحدة. عند أوّل تشغيل، تتولّى uv تحميل اعتماديات كاتب المعرّفة في PEP-723 تلقائياً.

    uv run scripts/build.py one-pager \
  --lang ar --title "كاتب يعمل" \
  --purpose "أوّل إنتاج"

    ستظهر لك رسالة تثبت نجاح الإنتاج. يحتوي مجلد الإخراج على ملف PDF، ونسخة HTML الأصلية بعد التصيير، وبيان manifest.md، ولقطة للرموز التصميمية، وملف .katib/run.json متوافق مع Obsidian.

تلميح

عيّن المتغيّر KATIB_OUTPUT_ROOT=/tmp/scratch قبل تشغيل الإنتاج ليُرسَل الإخراج إلى مسار مؤقّت — مفيد عند التجربة على القوالب.

الوحدة 3 · الهوية

ملفات الهوية — اجعل المستند يشبهك

كلّ إنتاج في كاتب يحترم تسلسلاً من أربع طبقات: رموز المجال الافتراضية ← ملف الهوية ← إعدادات سطر الأوامر ← التعديلات المباشرة داخل القالب. ملف الهوية نفسه مجرّد ملف YAML واحد يُغيّر الألوان والخطوط واسم الكاتب والشعار، دون المساس بأيّ قالب.

أهداف الوحدة
  • كتابة ملف هوية YAML في المسار ~/.katib/brands/<name>.yaml.
  • تمرير الخيار --brand <name> لأيّ عملية إنتاج.
  • فهم أيّ حقل يعود إلى القيمة الافتراضية للمجال عند عدم تعريفه في الهوية.

هويّة جاسم كمثال حيّ

هذا المستند نفسه منتج بهوية jasem — لوحة «الجمر الدافئ» بالكهرماني والبرتقالي، خطّ Cairo، ورقة فاتحة دافئة. قارِن اللوحة أدناه مع لوحة مجال tutorial الافتراضية (الفيروزي على ورقة دافئة) التي كنت سترى بها المستند لو لم تُحدِّد أيّ هوية.

ملف الهوية

# ~/.katib/brands/jasem.yaml
name: Jasem Al Neaimi
name_ar: جاسم النعيمي

identity:
  author_name: Jasem Al Neaimi
  email: jasem@jneaimi.com

colors:
  accent:            "#F59E0B"   # كهرماني
  accent_secondary:  "#FB923C"   # برتقالي
  page_bg:           "#FAFAFA"
  text:              "#18181B"

fonts:
  en:
    primary: "Newsreader"
    fallback: "Georgia, 'Times New Roman', serif"
  ar:
    primary: "Cairo"
    fallback: "Amiri, 'Traditional Arabic', serif"

logo:
  primary: logos/jasem.svg
  max_height_mm: 14

كيف تستخدمها

uv run scripts/build.py proposal \
  --domain business-proposal --lang ar \
  --brand jasem --with-cover \
  --title "عرض مشروع الرصد والمراقبة"
افعل احذف الحقول التي تريدها أن تعود إلى القيم الافتراضية للمجال — الهويات الجزئية مقبولة.
لا تفعل لا تُثبّت ألوان الهوية داخل القالب مباشرةً. استخدم var(--accent) ليعمل تبديل الهويات.
الوحدة 4 · التركيب

الأغلفة والتنسيقات والمخطّطات ولقطات الشاشة

يأتي كاتب بثلاثة محرّكات للأغلفة، وتنسيقين داخليّين، ومخطّطات SVG مدمجة بجودة طباعية، وخطّ إنتاج كامل للقطات الشاشة في الأدلّة التعليمية. هذه الوحدة هي مرجعك السريع للتركيب.

أنماط الأغلفة

minimalist-typographic
مجاني وقطعي، يعتمد CSS فقط. مناسب للمجالات التحريرية والخطابات الرسمية والمستندات الشخصية.
neural-cartography
يعتمد Gemini Nano Banana 2. بكلفة ~0.12 دولار للغلاف. ثقل مؤسسي — للعروض التجارية والخطابات الرسمية.
friendly-illustration
يعتمد Gemini Nano Banana 2. بكلفة ~0.12 دولار للغلاف. طابع دافئ وودود — للأدلّة التعليمية (وهو الغلاف الذي يزيّن هذا المستند بالذات).

خطّ إنتاج لقطات الشاشة

المخطّطات المدمجة

المخطّطات في كاتب عبارة عن SVG مدمج مباشرة داخل قالب HTML. يصيّر WeasyPrint الـ SVG أصلياً، فتبقى حادّة عند أي دقة طباعة، والنصّ داخلها قابل للاختيار والبحث، والألوان يمكنها اعتماد الهوية النشطة عبر استبدال القيم بمتغيّرات Jinja مثل رمز اللون اللوني الأساسي. كلّ مخطّط في هذا المستند منشأ بهذه الطريقة.

متى تستخدم ماذا

اعتمد SVG المدمج للمعمارية وتدفّق البيانات والرسوم البيانية — بجودة طباعية. استخدم مهارة /diagram (Excalidraw) للمحتوى التعبيري أو المرسوم يدوياً، لكنّه يتحوّل إلى نقطي عند الطباعة. أمّا لقطات الشاشة فتمرّ عبر سلسلة shot → annotate → frame.

الوحدة 5 · التحسّن الذاتي

Reflect — دع كاتب يتعلّم منك

كلّ إنتاج ناجح يُضيف سطراً إلى runs.jsonl. كلّ تصويب تُسجّله يذهب إلى feedback.jsonl. أنواع المستندات التي لا تنتمي لمجال قائم تهبط في domain-requests.jsonl. تقرأ أداة reflect.py الملفات الثلاثة وتَستخرج اقتراحات قابلة للتنفيذ — للقراءة فقط ودون تطبيق تلقائي.

أهداف الوحدة
  • فهم سجلّات الالتقاط السلبي الثلاثة.
  • تشغيل reflect.py مع نوافذ زمنية وترشيحات بحسب المجال.
  • معرفة متى تفعّل عتبة التكرار (≥ ثلاث مرّات) كلّ نوع من الاقتراحات.

أنواع الاقتراحات الثلاثة

لا يُفعَّل أيّ اقتراح إلّا عند تكرار النمط ثلاث مرّات فأكثر ضمن النافذة الزمنية النشطة (افتراضياً 30 يوماً). الرسم أدناه يوضّح سبب هذا الحدّ بالذات: أقلّ من ثلاث يعني ضجيجاً، عند الثلاث تبدأ الإشارة.

أنواع الاقتراحات

string-swap
تصويب متكرّر من صيغة إلى أخرى ضمن مجال ولغة واحدة. راجع القوالب وملف references/writing.<lang>.md.
new-domain-candidate
طلبات متكرّرة تُوجَّه إلى المجال الأقرب نفسه. فكّر في إضافة نوع مستند جديد أو مجال جديد.
unused-doc-type
نوع مستند مسجّل في styles.json لم يُنتَج ولو مرّة خلال النافذة. مرشّح للإلغاء — أكّد قبل الحذف.

جلسة نموذجية

uv run scripts/reflect.py                       # آخر 30 يوماً
uv run scripts/reflect.py --since 7d             # تضييق النافذة
uv run scripts/reflect.py --domain tutorial      # ترشيح حسب المجال
uv run scripts/reflect.py --propose              # كتابة ملف اقتراح Markdown
uv run scripts/reflect.py --json                 # مخرجات قابلة للمعالجة آلياً
أداة reflect.py للقراءة فقط عن قصد. لا تُعدّل القوالب ولا الرموز التصميمية ولا المراجع. التطبيق يدوي بعد المراجعة — هذا ما يحمي المهارة من الانجراف مع إشارة ضعيفة.
الوحدة 6 · الملحق

بطاقة الأوامر السريعة

كلّ ما يقدّمه كاتب من أدوات سطر أوامر، مجتمعاً في مكان واحد.

إنتاج مستند

build.py <doc-type> --domain <d> --lang <l> --brand <name>

إنتاج مع غلاف

أضف --with-cover --cover <style>. يتطلّب GEMINI_API_KEY.

جمع العربي والإنجليزي

مرّر نفس --slug للإنتاجين.

التحقّق من مجلد

build.py --verify <folder>

فحص القوالب

build.py --check — سريع، بلا إنتاج.

عرض الهويات

build.py --list-brands

توليد غلاف

cover.py --domain <d> --style <s> --out <path>

لقطة شاشة

shot.py web <url> --out <path>

إشارات وتسمية

annotate.py <png> --arrow x,y --label "..."

تأطير

frame.py <png> --style browser-safari

Reflect

reflect.py [--since Nd] [--domain <d>] [--propose] [--json]

متغيّرات البيئة

KATIB_OUTPUT_ROOT
تجاوز جذر المخرجات — مفيد للإنتاج المؤقّت.
KATIB_SLUG_OVERRIDE
فرض اسم مجلد محدّد للإنتاج التالي (أو استخدم --slug).
KATIB_IMAGE_MODEL
تجاوز نموذج Gemini للأغلفة (افتراضياً nano-banana-2).
KATIB_CACHE_DIR
مجلد ذاكرة التخزين المؤقّت للقطات.
KATIB_DEBUG=1
إخراج تفصيلي للإنتاج.
GEMINI_API_KEY
مطلوب للأغلفة التصويرية.

بنية الملفات التي يجب أن تعرفها

domains/<d>/
يحتوي tokens.json وstyles.json ومجلد templates/*.{en,ar}.html.
styles/layouts/
CSS المشترك (classic، workbook، cheatsheet).
brands/
ملفات الهوية المرفقة. ملفات المستخدم تعيش في ~/.katib/brands/.
scripts/
النصوص البرمجية: build.py، cover.py، shot.py، annotate.py، frame.py، reflect.py، memory.py.
references/
أدلّة التصميم والكتابة والإنتاج والمخطّطات ولقطات الشاشة — تُحمَّل حسب مستوى التفصيل المطلوب.

الخلاصة

بهذا تكون قد أغلقت الحلقة كاملةً: التثبيت، التوجيه إلى مجال، اختيار نوع مستند، تطبيق هوية، دمج مخطّطات ولقطات شاشة، والسماح لأداة reflect.py بكشف الأنماط مع تراكم استخدامك. كاتب مهارة حازمة عن قصد — وهذا بالضبط ما يجعل المخرجات جاهزة للطباعة دون أن تفكّر في قرارات التصميم مع كلّ إنتاج.

ما التالي

  • أنتج تسليماً حقيقياً — عرضك التجاري الأوّل لعميل، أو دليلاً كنت تؤجّله منذ فترة.
  • اكتب ملف هوية ثانياً لعميل، وسلّمه مسوّدات جاهزة للطباعة.
  • بعد عشرة أو عشرين إنتاجاً، شغّل reflect.py وشاهد أيّ الأنماط تطفو على السطح — هكذا يتعلّم كاتب ما تحتاجه أنت تحديداً.
