Module 1 · Foundations

Get set up

Start by installing the tools and confirming that your environment can reach every service you'll need in later modules.

Module objectives
  • Install and verify the core tool
  • Authenticate against the shared service
  • Run a smoke command that confirms everything is reachable

Steps

  1. Install the tool

    Run the installer appropriate for your OS. On macOS this is typically brew install <tool>; on Linux use your package manager of choice.

  2. Authenticate

    Open the tool and sign in with your workspace credentials. A success confirmation should appear within a few seconds.

  3. Verify connectivity

    Run tool status to confirm the tool can reach every backend it depends on. All checks should report green.

Tip

If status reports a warning, re-run with --verbose to see which specific dependency is degraded.

Module 3 · Going further

Handle edge cases and recover

Real work runs into friction. This module covers the three most common failure modes and a clean recovery path for each.

Module objectives
  • Identify the three failure patterns you're most likely to hit
  • Apply the standard recovery path without losing work
  • Know when to escalate vs. keep troubleshooting

Common failure modes

  • Permission denied: your role changed or a share expired — re-authenticate and retry.
  • Timeout: the backend is degraded — check the status page before further debugging.
  • Silent failure: the action appears to succeed but the UI doesn't refresh — a hard refresh almost always fixes it.
Watch out

If you see a red banner with no message body, the action wasn't saved — the error is upstream. Capture the request ID from the dev console before retrying.

When to escalate

Escalate to your team lead if the same failure persists across two full retries and a hard refresh. Include the request ID and the exact sequence of steps.
